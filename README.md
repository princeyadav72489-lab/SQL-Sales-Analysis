# Sales Data Analysis using SQL

## Objective
The objective of this project is to analyze the Sample Superstore sales dataset using SQL. The project demonstrates data exploration, filtering, aggregation, sorting, business analysis, and data validation using MySQL.

## Dataset
- Sample Superstore Dataset
- Records: 9994

## Tools Used
- MySQL Workbench
- GitHub Desktop
- GitHub

## SQL Concepts Covered
- SELECT
- WHERE
- GROUP BY
- ORDER BY
- LIMIT
- Aggregate Functions (SUM, COUNT, AVG, MIN, MAX)
- CASE Statement
- Data Validation

## Tasks Performed
- Imported the dataset into MySQL.
- Explored the table structure and sample records.
- Applied filters using the WHERE clause.
- Performed aggregations using GROUP BY and aggregate functions.
- Identified top products, categories and customers.
- Analyzed monthly sales trends.
- Checked for duplicate records.
- Validated data by checking missing values and sales statistics.

## Key Insights
- The dataset contains 9,994 records.
- Sales vary across different regions and categories.
- Technology and Furniture contribute significantly to total sales.
- Top customers generate a large share of total revenue.
- No missing values were found in the Sales, Category and Profit columns.
- Duplicate Order ID and Product ID combinations were identified for further review.
